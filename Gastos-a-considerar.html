Pagos

Dos usos distintos de Stripe

Fáciles de confundir porque ambos son "Stripe", pero resuelven cosas distintas.

Stripe Billing — para que tú cobres la membresía

Suscripciones recurrentes de MTA a tus clientes. Sin costo de licencia, solo comisión por transacción (~3.6% + $3 MXN en México). Esto es lo que necesitas para vender por membresía.

Stripe Terminal — feature dentro del producto

Lector físico de tarjeta para que tus clientes (salones, talleres) cobren en persona a sus propios clientes. Hardware aparte (~$59–300 USD/lector). Opcional, se puede posponer para después del lanzamiento.

Dominio

Namecheap / Cloudflare

~$12–15 USD/año. Sin renta adicional de SSL — se cubre gratis con Let's Encrypt o el certificado administrado del proveedor.

Legal & facturación

No es licencia de software, pero aplica

Para cobrar suscripciones formalmente en México necesitas RFC con actividad empresarial (persona física o moral), términos de servicio y aviso de privacidad. No es parte del stack técnico, pero es requisito para vender de forma legal.

Checklist para arrancar

  • Decidir IDE antes de que venza el trial de Rider (30 días): comprar licencia Personal o migrar a VS Community
  • Levantar VPS económico (Hetzner / Railway / Render) para reemplazar el hosting Azure planeado originalmente
  • Crear Postgres administrado en Neon o Supabase (tier free)
  • Registrar dominio
  • Configurar Stripe Billing cuando esté listo el flujo de suscripción de MTA
  • Dejar Blob Storage, Stripe Terminal y email real (Azure Communication / Resend) para cuando el producto tenga esos flujos construidos
  • Resolver RFC / facturación antes de cobrar el primer cliente real
Con esto puedes estar operando por menos de $20 USD/mes hasta tener clientes pagando.
