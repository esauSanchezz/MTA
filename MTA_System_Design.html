¿Qué es MTA?
Una plataforma que se adapta a tu negocio
MasterTrackAgenda es un sistema de gestión inteligente multi-tenant. Un solo backend sirve a distintos tipos de negocio — salón de uñas, taller mecánico, salón de fiestas — con agenda, trabajadores, inventario, ventas, POS y reportes integrados. Cada negocio ve únicamente sus propios datos.

    🗄️ Una sola base de datos PostgreSQL

    EF Core aplica WHERE TenantId = '...' automáticamente en cada query via Global Query Filters.
    Cada negocio solo ve sus propios datos. Sin código extra en cada controller.

    📅 Flujo 1: Agendar y atender una cita
    1

    Cliente solicita cita

    Se crea en ScheduleTasks con Type=Appointment, Status=Scheduled. Se asigna al empleado disponible según su WorkSchedule.

    2

    Recordatorio automático (Durable Function)

    24h antes → SMS/email al cliente → Status=Confirmed. Si no confirma 15 min después → alerta al empleado.

    3

    Se atiende la cita

    Empleado cambia a Status=InProgress → termina → Status=Completed.

    4

    Cobro en POS

    Se crea un Sale vinculado al ScheduleTask, con SaleItems (servicio + productos adicionales). Se elige método de pago: efectivo, tarjeta (Stripe Terminal) o transferencia.

    5

    Recibo generado automáticamente

    Se crea un Receipt con número consecutivo MTA-2026-00001, PDF en Azure Blob Storage, y se envía por email al cliente.

    📦 Flujo 2: Control de inventario
    1

    Compra a proveedor

    Se crea una PurchaseOrder. Al marcarla Status=Received, el sistema suma automáticamente el stock en cada Product.

    2

    Venta descuenta stock

    Al procesar una Sale con productos, el sistema resta el Stock. Si Stock <= MinStock → Durable Function envía alerta al admin.

    3

    Re-order automático

    Si tras 24h no se ha creado una orden de compra, Durable Function re-notifica. Máximo 3 veces.

    💰 Flujo 3: Corte de caja automático
    1

    Trigger automático (11:55pm)

    Durable Function se activa cada noche por tenant.

    2

    Agrega ventas del día

    Suma todas las Sales del día por método de pago. Resta gastos y compras.

    3

    Genera PDF y notifica al Owner

    Crea el registro en CashRegisterCuts, genera PDF en Azure Blob Storage y lo envía por email al dueño del negocio.

    🏢 Flujo 4: Onboarding de nuevo negocio
    1

    Registro del negocio

    Se crea el Tenant con tipo de negocio, nombre y slug único.

    2

    Carga catálogo default

    Según el BusinessType, Durable Function carga servicios predefinidos, categorías de productos y configuración base.

    3

    Bienvenida + seguimiento

    Crea el usuario Owner, envía email de bienvenida. Si no inicia sesión en 3 días → recordatorio automático.

    Azure Durable Functions
    Workflows automáticos del sistema
    Los Durable Functions manejan procesos largos o multi-paso que no pueden vivir en una simple llamada HTTP. Son stateful — sobreviven reinicios del servidor.

    🔔 DF-01 — Recordatorio de Cita

    Se activa cuando se agenda una cita. Maneja toda la comunicación pre-cita.

    → Cita agendada (trigger)
    → Espera T-24h → SMS/email cliente
    → Espera T-2h → 2do recordatorio
    → T+15min sin atender → alerta empleado
    → Sin confirmar → Status=NoShow

    📦 DF-02 — Alerta de Stock Mínimo

    Se activa cuando una venta descuenta inventario.

    → Venta procesada (trigger)
    → Verifica Stock vs MinStock
    → Si bajo → notifica al admin
    → Espera 24h → si sigue bajo, re-notifica
    → Máximo 3 notificaciones

    💰 DF-03 — Corte de Caja Automático

    Timer diario. Genera el resumen financiero del día.

    → Timer 11:55pm por tenant
    → Agrega ventas del día
    → Calcula gastos y compras
    → Genera PDF del corte
    → Email al Owner

    🏢 DF-04 — Onboarding de Negocio

    Se activa al registrar un nuevo tenant en el sistema.

    → Registro nuevo tenant
    → Crea datos base del tenant
    → Carga catálogo default por tipo
    → Crea usuario Owner
    → Email bienvenida + si no inicia sesión en 3 días → recordatorio

    💳 DF-05 — Pago con Retry

    Maneja el cobro con Stripe con reintentos automáticos.

    → Intento de cobro Stripe
    → Si falla → espera 30s → reintento
    → Máximo 3 intentos
    → Si sigue fallando → alerta al cajero
    → Si éxito → genera recibo → email
